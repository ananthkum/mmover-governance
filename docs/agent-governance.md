# Microsoft 365 Copilot Agent Governance

Purpose:
- Define the governance controls for building and publishing a Microsoft 365 Copilot agent using the upgraded mMoveR governance.

Applies to:
- Agent Builder in Microsoft 365 Copilot.
- Copilot Studio when advanced capabilities or external actions are required.

Mandatory controls:
- Business purpose is explicit and narrow.
- Instructions define scope, allowed sources, refusal behavior, and escalation behavior.
- Knowledge sources are approved and documented.
- Sharing scope is least-privilege.
- Billing model is confirmed before using tenant data in Copilot Chat scenarios.
- Publish approval requires PO and admin sign-off.
- Validation uses representative prompts for positive, negative, edge, and abuse cases.
- Versioned changelog maintained for every material instruction or knowledge change.
