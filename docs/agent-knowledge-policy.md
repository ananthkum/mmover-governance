# Agent Knowledge Policy

Knowledge source rules:
- Each source must have an owner, purpose, and review date.
- Prefer approved SharePoint locations and controlled web sources.
- Do not add broad sources when a narrow source can satisfy the use case.
- Record sensitivity and sharing impact for each source.
- Remove stale or duplicate sources.

Validation rules:
- Test source grounding with prompts that require precise retrieval.
- Test access-denied behavior with users outside the allowed scope.
