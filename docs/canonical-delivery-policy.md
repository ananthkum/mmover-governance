# Canonical File Delivery Policy (NO DIFFS)

Rules:
- Always deliver complete canonical files for any changed governance or artifacts.
- Never deliver diffs or partial snippets for governance or docs.
- Never include cut-off output markers or stub text.
- If long, deliver as downloadable ZIP containing full files.
- For mMoveR code, use canonical rewritten files or explicit search/replace only when the change is truly minimal.

Immediate packaging:
- Any governance change triggers immediate downloadable ZIP of full canonical governance set.
